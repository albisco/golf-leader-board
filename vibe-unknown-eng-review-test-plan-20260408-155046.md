## Framework
pytest + httpx (async FastAPI test client)
WebSocket tests: pytest-asyncio + starlette TestClient

## Affected Pages/Routes

- `POST /events` — create event with holes auto-generated
- `POST /events/{id}/start` — draft → active transition
- `POST /events/{id}/close` — active → closed transition
- `POST /scores` — score submission (upsert on hole)
- `GET /leaderboard/{event_id}` — live leaderboard data
- `POST /chat` — chat message creation
- `POST /auth/magic-link` — send magic link email
- `GET /auth/verify?token=...` — validate magic link, set session
- `GET /score/{scorer_token}` — scorer interface (auth by token)
- `WS /ws/{event_id}` — WebSocket leaderboard + chat updates

## Key Interactions to Verify

1. Score upsert: submitting a score for the same hole twice updates the existing
   row; edit_count increments; updated_at changes
2. Leaderboard mid-round sort: groups with more holes completed rank higher;
   within equal holes_played, lower gross ranks higher
3. Leaderboard final sort: when holes_played == event.hole_count, sort by
   net_total ASC then gross_total ASC (tiebreaker)
4. WebSocket broadcast fires after score submission — all connected clients
   receive updated leaderboard HTML
5. Closed event rejects score submission with 403
6. Magic-link expires after 24 hours; re-use within expiry still works
7. Organizer delete chat message — removed from event-wide feed

## Edge Cases

- `hole_count=16` (non-standard event): 16 holes created, leaderboard shows
  final results when all 16 submitted (not 18)
- Two groups with identical net total AND identical gross: shared placing,
  no crash, no arbitrary ordering
- Incomplete group (< hole_count scores): ranked below all complete groups,
  shows "(N holes)" in leaderboard row
- Invalid scorer token URL: returns 404, not 500
- Score with negative gross_score or null: returns 422
- WebSocket reconnect: client receives full leaderboard snapshot on reconnect

## Critical Paths (E2E — use pytest-asyncio with real DB)

1. **Full scorer flow:** POST /events → POST /events/{id}/start → GET
   /score/{scorer_token} → POST /scores (×hole_count) → GET /leaderboard
   → leaderboard shows correct net total
2. **Live broadcast:** Two WS clients connected → score submitted →
   both clients receive updated leaderboard HTML within 1s
3. **Chat flow:** WS connected → POST /chat → WS client receives chat
   message HTML fragment
